🔍 案发现场在哪里？
在你的 MEM.sv 中，你对 addr 这个信号同时做了两次规定：
第一次（第 57 行左右，并发赋值）：
code
Verilog
assign addr = ex_mem_bus.w_data; 
// 硬件意义：焊死一根线，把 w_data 永远连在 addr 上。
第二次（第 134 行左右，过程赋值）：
在 always_comb 的 else 分支里：
code
Verilog
} else begin
        mem_wb_bus.ram_data = 32'b0;
        addr = 32'b0;  // <---- 就是这句惹的祸！
    end
    
硬件意义：如果 memory_re 是 0，强制把 addr 接到地（0V）。
为什么报错？
想象一下实际的电路：你用一根导线（assign）把 addr 连到了 ALU 的输出端；但同时，你又在电路的另一头加了个开关（always_comb），说“不读内存时，我要把它接地”。这会导致物理电路上的短路冲突（多驱动源）。Vivado 综合器不知道该听谁的，所以直接报错拦截。
